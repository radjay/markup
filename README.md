# Markup

A native desktop app for reviewing markdown documents generated by AI coding agents. Open a plan, leave comments on any section, save — and the agent picks up your feedback.

## The Problem

AI coding agents like Claude Code generate markdown plans and documents. Reviewing them is painful: you're stuck reading raw files in a terminal, switching to a text editor to leave notes, then copy-pasting feedback back into the conversation. There's no tool that closes the loop between agent output and human feedback.

## How Markup Works

1. **Agent generates a markdown plan** (e.g., `docs/plans/feature-plan.md`)
2. **You open it in Markup** — rendered beautifully, not raw text
3. **Click any block to leave a comment** — headings, paragraphs, tables, code blocks, lists
4. **Save** — comments are embedded as invisible HTML comments in the file
5. **Agent reads the file** and addresses your feedback, then removes resolved comments

Comments use the `@markup` format — invisible in any standard markdown renderer, machine-parseable by agents:

```html
<!-- @markup {"id":"...","type":"inline","anchor":"h2:Architecture"} This won't scale. Use a modular monolith instead. -->
```

## Features

**Review**
- Beautiful rendered markdown with Literata headings, Inter body text, JetBrains Mono code
- Click any block (heading, paragraph, table, code, list, blockquote) to add inline comments
- Document-level comments panel for general feedback
- Comments saved as HTML comments — invisible in other renderers

**Navigate**
- Multi-folder workspace — add project folders, persisted across sessions
- Tree view and Recently Updated view (sorted by last modified, live-refreshing)
- Tabbed editor with VS Code-style preview mode (single-click = preview, double-click = pin)
- Document outline in the right panel

**Edit**
- Toggle between Review and Edit mode (Cmd+E)
- CodeMirror 6 with markdown syntax highlighting
- File watching — detects external changes with reload prompt

**Agent Integration**
- Claude Code plugin included (`plugin/CLAUDE.md`)
- Teaches agents how to parse `@markup` comments and address feedback
- Review status in frontmatter (`markup_status: changes_requested`)

## Comment Format

### Inline comments

Inserted after the block they reference:

```markdown
## Architecture

The system uses microservices.
<!-- @markup {"id":"c1","type":"inline","anchor":"h2:Architecture","ts":"2026-03-28T10:30:00Z"} Consider a modular monolith instead. -->
```

### Document-level comments

Appended at the end of the file:

```markdown
<!-- @markup-doc-comments
{"id":"d1","type":"document","ts":"2026-03-28T10:35:00Z","body":"Overall the plan needs more detail on failure scenarios."}
-->
```

### Review metadata

Added to YAML frontmatter:

```yaml
markup_reviewed: true
markup_reviewed_at: "2026-03-28T10:36:00Z"
markup_status: changes_requested
```

## Claude Code Plugin

To teach Claude Code how to handle Markup comments, add to your project's `CLAUDE.md`:

```markdown
When you encounter HTML comments matching `<!-- @markup {...} ... -->`,
these are review comments from the Markup editor. Address each comment
by updating the relevant section, then remove the comment block.
```

Or copy the full instructions from `plugin/CLAUDE.md`.

## Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| Cmd+O | Open file |
| Cmd+S | Save |
| Cmd+E | Toggle Review/Edit mode |
| Cmd+W | Close tab |
| Cmd+Enter | Submit comment |
| Escape | Close comment panel |
